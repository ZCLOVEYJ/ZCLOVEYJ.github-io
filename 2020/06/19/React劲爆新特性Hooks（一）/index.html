    预备知识

    工具类

    node

    作为前端工程师,肯定都听说过node，全称是node.js，以往我们只能在浏览器中运行JS代码，要在本地执行一些逻辑，只好求助一些其他的编程语言，比如Java、php，node的出现完全改变了这一局面。
    我们知道前端工程师最常用的开发浏览器是谷歌的chrome，而chrome所使用的js引擎是他们自己开发的v8，那么node其实就是在v8的基础上增加了一些本地API，比如基于异步事件的文件系统，进程和网络，实现了让js在本地运行的数据源，比如开启HTTP服务端口，读写文件等能力，这些在之前都是不可能实现的。

    npm

    全称node package manager。

    webpack

    我们需要的很多第三方模块由npm下载和管理，显然我们不可能原封不动发送到浏览器运行，1.因为这些模块规范无法直接被浏览器环境直接识别，2.是代码必须被合并提高浏览器的加载速度。webpack就是用来做这件事的。

    eslint

    eslint是javascript环境中最受欢迎的代码风格校验工具，通过数百条规则的配置，我们能够检查代码中不符合规范的地方，甚至能自动修复，eslint的规则是可以自定义的。

    prettier

    虽然eslint也能够自动处理一些代码格式问题，但是整体功能比较弱，特别是对其他语言的语法无能为力，所以引出Facebook公司的另一个有力工具，叫做prettier。顾名思义，它也是用来处理代码格式问题的，不同于eslint``prettier提供的选项非常有限，或者说prettier的行为更有一些武断的意味，它只按照自己的预测行为来运行，并不给你很多的调整机会。此外，prettier还天生提供了对多种语言语法的支持，后面我们将配置来让他帮我们自动的美化我们的代码。

    总结

    以上就是我们所依赖和使用的几个工具，相对来讲它们都非常基础，越是基础越是重要，希望我对他们都能够熟悉起来。

    概念类

    接下来我们要了解的是前端开发中的几个概念，它们分别代表了就一系列显著特征的现象。

    SPA/MPA

    SPA全称single page application，俗称单页应用，典型的特征是用仅仅一个HTML页面，就能承载一个业务的所有逻辑。通常使用异步接口与服务端通信，并在前端渲染HTML页数，而不是在服务端直接渲染HTML约束。SPA的概念源于业界在前后端分离的开发模式上的探索，能够实现前端开发和后端开发的高度结合。近年来SPA已经成为了大多数业务的首选技术方案，那么对于稍微复杂一点的业务，可能把所有业务都放到一个HTML页面中，显得略微臃肿，那么就衍生出了另一种模式叫做MPA multiple pages up，多页应用，除了它包含多个页面，而不是仅仅一个页面对外，它和SPA并没有本质区别。

    PWA

    PWA全称是progressive web APP，俗称渐进式网络应用。它和我们传统的业务有什么不同呢？从名字上来看，既然是渐进式，说明它也可以退化。没错，PWA退化的模式就是我们常规的浏览器HTTP访问页面的模式，那么什么时候才退化呢？答案是浏览器不支持PWA所需要的API的时候，如果浏览器支持，那么我们的页面将具备一个重要的能力，那就是可以控制包括js在内的静态资源缓存，这就意味着即便我们的设备没有联网，我们依旧可以使用缓存来运行页面，这就给我们提供了强大的离线访问能力。即便不在离线模式下，缓存也将大大提升了页面的载入速度。可以说PWA是前端领域近年来的革命性进化，到目前为止只有少部分老旧浏览器还不支持，特别是在移动端，IOS和安卓都已经标配了对pwa的支持。
